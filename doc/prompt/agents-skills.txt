提交代码流程：
1. 确保远程仓库是 xx 域名下，否则退出执行，只能提交代码到 xx 域名仓库
2. 确保提交的变更中不包含敏感信息，比如：账号，密码，密钥等
3. 如果当前分支在 master 过 main 上，则拉取最新的 master 或 main 分支，然后基于 master 或 main 分支创建 feature 分支，feature 分支命名规范如下：feature/CH_000-{NAME}，替换{NAME}为具体有意义的名称，例如 fix-timeout
4. 如果当前分支已经是 feature 分支了，则不需要创建新分支了
5. 如果要提交的变更内容比较多，按逻辑拆分多个变更提交，确保每个变更提交文件数量不超过 5 个
6. 提交变更时需要填写 comments，comments 填写内容规范如下：必须有意义的描述，长度不少于二十个字符
